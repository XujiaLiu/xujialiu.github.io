git配置

查看git配置 

# 查看git配置信息
git config --list

# 查看git系统配置信息
git config --system --list
# 配置文件在...\Git\etc\gitconfig

# 查看当前用户git配置
git config --global --list
# 配置文件在...\user\.gitconfig

修改git配置

  1. 直接修改用户文件夹下面的.gitconfig文件
[user]
    name = xujialiu
    email = xujialiuphd@gmail.com
  1. 使用命令行
git config --global user.name "xujailiu"
git config --global user.email "xujialiuphd@gmail.com"

git理论

git区域

git有4个工作区域，包括3个本地工作区域和1个远程工作区域

  • Workspace：工作区，平时存放项目代码的地方
  • Index / Stage：暂存区，用于临时存放改动文件，事实上它只是一个文件，保存即将提交到文件列表信息
  • Repository：仓库区（或本地仓库），就是安全存放数据的位置，这里面有你提交到所有版本的数据。其中HEAD指向最新放入仓库的版本
  • Remote：远程仓库，托管代码的服务器，可以简单的认为是你项目组中的一台电脑用于远程数据交换

git工作流：

１、在工作目录中添加、修改文件；
２、将需要进行版本管理的文件加入git add暂存区域；
３、将暂存区域的文件提交git commit到git仓库。

git仅需机组下面6个命令加上git init

git文件操作

  • Untracked: 未跟踪, 此文件在文件夹Workspace中, 但并没有加入到git库, 不参与版本控制. 通过git add状态变为Staged.
  • Unmodify: 文件已经入库, 未修改, 即版本库中的文件快照内容与文件夹中完全一致. 这种类型的文件有两种去处, 如果它被修改, 而变为Modified. 如果使用git rm移出版本库, 则成为Untracked文件
  • Modified: 文件已修改, 仅仅是修改, 并没有进行其他的操作. 这个文件也有两个去处, 通过git add可进入暂存staged状态, 使用git checkout 则丢弃修改过, 返回到unmodify状态, 这个git checkout即从库中取出文件, 覆盖当前修改 !
    执行git commit则将修改同步到库中, 这时库中的文件和本地文件又变为一致, 文件为Unmodify状态. 执行git reset HEAD filename取消暂存, 文件状态为Modified

git常用命令 

# 查看所有文件状态
git status

# 在当前目录新建一个git代码库即.git文件夹
git init

# 添加当前所有文件到暂存区
git add .

# 将文件提交到本地仓库
git commit -m "message for commit"

# 克隆远程仓库到本地
git clone git@github.com:XujiaLiu/test_git.git

.gitignore文件忽略某些文件 

# 注释
*.txt        # 忽略所有 .txt结尾的文件
!lib.txt     # 忽略所有, lib.txt除外
/temp        # 忽略项目根目录下的temp文件
build/       # 忽略build/目录下的所有文件
doc/*.txt    # 会忽略 doc/notes.txt

git分支常用命令 

# 查看本地分支
git branch

# 查看远程分支
git branch -r

# 本地添加dev分支，此时仍然停留在当前分支而不是dev分支
git branch dev

# 合并指定分支到当前分支
git merge [branch]

# 删除分支
git branch -d [branch]

远程和本地同步

在gitee或github创建项目，然后clone到本地，把.git文件夹和其他的文件放到需要同步的文件即可

git clone git@github.com:XujiaLiu/test_git.git

或者在gitee或github创建完项目，按照指示操作即可
