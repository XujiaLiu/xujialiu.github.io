require和library的区别

  1. 错误处理：如果尝试加载不存在的包，library() 函数会产生一个错误，而 require() 函数会产生一个警告。在执行 library() 后，如果包不存在，脚本将停止执行；而在执行 require() 后，脚本将继续执行。因此，如果您希望在缺少包的情况下继续执行脚本，可以使用 require() 函数。
  2. 返回值：library() 函数没有返回值，而 require() 函数返回一个逻辑值。如果包加载成功，require() 返回 TRUE，否则返回 FALSE。这使得您可以根据包是否加载成功来决定执行不同的代码逻辑。

单模式数据结构

class(), typeof(), mode() 函数

  1. class(): 这个函数返回的是对象的类（class）。这是最高级的类型信息，可以用来查看对象的面向对象类型。例如，对于一个数据框，class()返回的是"data.frame"，这表示这个对象是数据框类的一个实例。类信息常常用于方法派发，决定对对象应用哪个函数或操作。

  2. typeof(): 这个函数返回对象的内部存储类型。这对于R的原始数据类型是非常准确的，例如"integer"，"numeric"（用于浮点数），"character"，"logical"，"complex"，"raw"等。对于R的更高级的数据结构，例如列表（list）或数据框（data.frame），typeof()返回的是这些结构的底层类型，例如，对于列表，它返回"list"，对于数据框，它返回"list"（因为数据框在底层其实就是一个特殊的列表）。

  3. mode(): 这个函数返回的是对象的模式（mode）。这是一种更抽象，更少细节的类型信息，常用的模式包括"numeric"，"character"，"list"等。对于数据框，mode()返回的是"list"。

class() 函数的输出：

  1. "data.frame"：数据框对象。
  2. "factor"：因子对象。
  3. "Date"：日期对象。
  4. "POSIXct" 和 "POSIXlt"：表示日期和时间的对象。
  5. "ts"：时间序列对象。
  6. "matrix"：矩阵对象。
  7. "array"：数组对象。
  8. 对于 S3 和 S4 对象，class() 会返回这些对象的类名。

mode() 函数的输出：

  1. "logical"：逻辑值，例如 TRUE 或 FALSE。
  2. "numeric"：数值类型，包括整数和浮点数。
  3. "complex"：复数值，包含实部和虚部。
  4. "character"：字符值，如字符串。
  5. "list"：列表对象，可以包含不同类型的元素。
  6. "function"：函数，包括用户定义的和内置函数。
  7. "language"：语言对象，包括符号、表达式和其他表示 R 代码的结构。
  8. "NULL"：NULL 对象，表示空值。
  9. "S4"：S4 对象，这是一种使用 S4 类系统的对象。

typeof() 函数返回对象的内部类型

  1. "logical"：逻辑值，例如 TRUE 或 FALSE。
  2. "integer"：整数值。
  3. "double"：双精度浮点数，通常称为数值（numeric）。
  4. "complex"：复数值，包含实部和虚部。
  5. "character"：字符值，如字符串。
  6. "list"：列表对象，可以包含不同类型的元素。
  7. "NULL"：NULL 对象，表示空值。
  8. "closure"：闭包，通常是用户定义的函数。
  9. "builtin" 或 "special"：内置函数或特殊函数，这些是 R 语言核心中实现的基本函数。
  10. "environment"：环境对象，用于存储变量和函数定义。
  11. "expression"：表达式，由一组表达式组成，可以作为整体进行求值。
  12. "promise"：承诺（promise）对象，用于延迟求值，通常在函数调用中使用。
  13. "symbol"：符号，表示变量名或函数名。
  14. "pairlist"：配对列表，这是一种较早的数据结构，现在已经很少使用。
  15. "externalptr"：外部指针，指向 R 语言之外的对象，通常用于与其他编程语言的接口。
  16. "bytecode"：字节码，用于存储编译后的 R 函数。
  17. "S4"：S4 对象，这是一种使用 S4 类系统的对象。
  18. "weakref"：弱引用，用于管理指向其他对象的引用，而不会阻止垃圾收集器回收这些对象。

NA与NULL与NaN

  • NULL：NULL 是一个特殊的空值，表示没有对象或数据存在。通常，NULL 用于初始化变量或表示空列表、空向量等。NULL 是一个长度为零的对象，它在 R 中具有唯一的类（即 class(NULL) 的输出为 "NULL"）。

  • NaN（Not a Number）：表示数值计算中产生的特殊浮点值，表示结果未定义或无法表示的数值。例如，0/0 或无穷大减去无穷大等操作会得到 NaN。

aggregate() 函数

对数据进行分组汇总的函数

## S3 method for class 'data.frame'
aggregate(x, by, FUN, ..., simplify = TRUE, drop = TRUE)

## S3 method for class 'formula'
aggregate(x, data, FUN, ...,
          subset, na.action = na.omit)

aggregate() 函数接受以下参数：

  1. x: 一个 R 对象，通常是一个数据框（data.frame），但也可以是矩阵或数组。
  2. by: 一个包含分组变量的列表，用于将数据分成不同的组。
  3. FUN: 要应用于每个分组的聚合函数。它可以是标准的 R 函数（如 sum、mean、length 等）或用户自定义的函数。
  4. ...: 额外的参数传递给聚合函数 FUN。
  5. simplify: 一个逻辑值，表示是否尝试简化结果。默认为 TRUE。如果设置为 FALSE，则结果将始终为列表。
  6. drop: 一个逻辑值，表示是否应删除没有观测值的组合。默认为 TRUE。
# 公式
aggregate(Wind ~ Month, data = airquality, FUN = median)

# 多变量汇总
aggregate(Wind ~ Month + cut(Temp, 2), data = airquality, FUN = median)

# 多列汇总
aggregate(cbind(Wind, Ozone) ~ Month, data = airquality, FUN = median)

# 多个返回值
aggregate(cbind(Wind, Ozone) ~ Month + cut(Temp, 2), data = airquality,
		  FUN = range, na.rm = TRUE)    # 注意na.rm这个关键字参数是传给range的

# 指定列 
aggregate(list(aveWind = airquality$Wind), list(Month = airquality$Month)

# 多变量汇总
aggregate(list(aveWind = airquality$Wind),
		  list(Month = airquality$Month, TempGroup = cut(airquality$Temp,2)), 
		  median)

# `多列汇总`
aggregate(list(aveWind = airquality$Wind, aveOzone = airquality$Ozone),
		  list(Month = airquality$Month), 
		  median, na.rm = TRUE)

# 多个返回值
aggregate(list(Wind = airquality$Wind),
		  list(Month = airquality$Month), range)

lobstr包

lobstr包是一个用于深入了解R语言中对象结构和内存分配的工具包。它提供了一系列函数，帮助用户更好地了解R中的数据对象及其内部工作原理。lobstr包的主要目标是提供对R对象内部结构的直观和清晰的表示。
这个包中的一些主要函数包括：

  1. obj_size()：计算对象的内存占用大小。
  2. ref()：显示对象的引用计数，即有多少对象引用了同一块内存。
  3. obj_addr()：返回对象的内存地址，以便了解它们是否指向相同的内存空间。
  4. ast()：显示R表达式的抽象语法树（Abstract Syntax Tree），有助于理解代码的解析和执行过程。
  5. tree()：展示数据对象的树形结构，以便更好地理解复杂数据类型（如列表、数据框等）的内部结构。
  6. unclass()：用于删除对象的类属性，将其转换为底层的数据类型。
    lobstr包对于那些想要更深入了解R语言内部工作原理的用户非常有用，特别是在调试和优化代码性能方面。

参数：

  2. col：要展开的列表列。当选择多个列时，相同行的值将循环扩展到它们的公共大小。
  3. values_to：用于存储展开值的列名。如果在 col 中指定了多个列，这也可以是包含 "{col}" 的 glue 字符串，以提供列名模板。默认值 NULL 会使输出列具有与输入列相同的名称。
  4. indices_to：用于存储值的内部名称或位置（如果未命名）的列名。如果在 col 中指定了多个列，这也可以是包含 "{col}" 的 glue 字符串，以提供列名模板。默认值 NULL 会使输出列具有与 values_to 相同的名称，但后缀为 _id。
  5. indices_include：指定是否添加索引列的单个逻辑值。如果任何值具有内部名称，索引列将是这些名称的字符向量，否则将是位置的整数向量。如果为 NULL，则在提供任何值具有内部名称或提供 indices_to 时，默认为 TRUE。如果提供了 indices_to，那么 indices_include 不能为 FALSE。
  6. keep_empty：默认情况下，每个列表元素的输出将为一行。这意味着，如果有大小为 0 的元素（如 NULL 或空数据框或向量），则整个行将从输出中删除。如果要保留所有行，请使用 keep_empty = TRUE 将大小为 0 的元素替换为缺失值的单行。
  7. names_repair：用于检查输出数据框是否具有有效名称。必须是以下选项之一：
    • "minimal"：除基本存在外，没有名称修复或检查。
    • "unique"：确保名称唯一且不为空。
    • "check_unique"：（默认）不进行名称修复，但检查它们是否唯一。
    • "universal"：使名称唯一且符合语法。
    • 函数：应用自定义名称修复。
    • 公式：一个符合 purrr 风格的匿名函数（参见 rlang::as_function()）。
  8. simplify：如果为 TRUE，将尝试将长度为 1 的向量列表简化为原子向量。还可以提供一个包含 TRUE 或 FALSE 的命名列表，声明是否尝试简化特定列。如果提供了命名列表，默认值为任何未指定列的 TRUE。
  9. ptype：可选的，原型的命名列表，声明每个组件期望的输出类型。或者，可以提供一个空的原型，它将应用于所有组件。如果要在简化时检查每个元素是否具有您期望的类型，请使用此参数。
    如果已指定 ptype，但 simplify = FALSE 或无法简化，则将返回一个 list-of 列，每个元素的类型为 ptype。
  10. transform：可选的，应用于每个组件的转换函数的命名列表。或者，可以提供一个将应用于所有组件的单个函数。如果要在提取时转换或解析各个元素，请使用此参数。
    当提供 ptype 和 transform 时，将在 ptype 之前应用 transform。

pacman 包是 R 语言中的一个非常实用的包管理器，它能帮助用户更方便地安装、加载、卸载和更新 R 包。pacman 的名字源于 "Package Management"（包管理）的缩写。它提供了许多实用函数，以简化包管理任务。

# 这将从当前 R 会话中卸载 `ggplot2` 和 `dplyr`。
  1. p_remove：卸载并从系统中删除一个或多个 R 包。例如：

ggsave()函数

用于保存 ggplot 图形或其他 grid 对象
它默认保存最后显示的图形，并使用当前图形设备的大小。此外，它还会根据文件扩展名猜测图形设备的类型。

ggsave(
  filename,
  plot = last_plot(),
  device = NULL,
  path = NULL,
  scale = 1,
  width = NA,
  height = NA,
  units = c("in", "cm", "mm", "px"),
  dpi = 300,
  limitsize = TRUE,
  bg = NULL,
  ...
)

参数解释：

  • filename：要在磁盘上创建的文件名。
  • plot：要保存的图形，默认为最后显示的图形。
  • device：要使用的设备。可以是设备函数（如 png），或者是 "eps"、"ps"、"tex"（pictex）、"pdf"、"jpeg"、"tiff"、"png"、"bmp"、"svg" 或 "wmf"（仅限 Windows）。
  • path：要将图形保存到的目录的路径：将路径和文件名组合以创建完全限定的文件名。默认为工作目录。
  • scale：乘法缩放因子。
  • width、height、units：图形尺寸（单位为 "in"、"cm"、"mm" 或 "px"）。如果未提供，将使用当前图形设备的大小。
  • dpi：图形分辨率。也可以接受字符串输入："retina"（320）、"print"（300）或 "screen"（72）。仅适用于光栅输出类型。
  • limitsize：当为 TRUE（默认值）时，ggsave() 不会保存大于 50x50 英寸的图像，以防止指定像素尺寸的常见错误。
  • bg：背景颜色。如果为 NULL，使用图形theme中的 plot.background 填充值。
  • ...：其他参数传递给图形设备函数，由 device 指定。

aes()函数

定义图形的美学映射（aesthetic mappings）。美学映射是将数据的变量与图形属性（如颜色、形状、大小等）关联起来的方式。

aes(x, y, ...)
  • x：x 轴上的变量。
  • y：y 轴上的变量。
  • color：根据变量改变点或线的颜色。
  • fill：根据变量改变填充颜色（适用于柱状图、箱线图等）。
  • shape：根据变量改变点的形状（适用于散点图）。
  • size：根据变量改变点或线的大小。
  • linetype：根据变量改变线条类型（适用于线图）。

ggplot2

  1. 数据（Data）：在ggplot2中，数据通常以数据框（data.frame）的形式提供。你可以使用自己的数据集，也可以使用R中内置的数据集。数据框中的每个变量都可以在图形中以某种形式表示。
  2. 映射（Aesthetics）：映射是指将数据框中的变量与图形的视觉属性（如坐标、颜色、大小等）相关联。例如，你可以将一个变量映射到x轴，另一个变量映射到y轴，还可以将第三个变量映射到点的颜色。
  3. 几何对象（Geoms）：几何对象是图形中实际绘制的元素，例如点、线、柱等。ggplot2中有许多几何对象，如geom_point（散点图）、geom_line（折线图）、geom_bar（柱状图）等。每个几何对象都有其特定的函数，用于在图形中添加相应的元素。
  4. 标度（Scales）：标度控制着如何将数据值映射到图形的视觉属性。例如，颜色标度（如scale_color_gradient）可以将连续变量映射到一系列颜色，而大小标度（如scale_size_continuous）可以将连续变量映射到点的大小。通过自定义标度，你可以更好地控制图形的外观。
  5. 坐标系（Coordinates）：坐标系定义了如何将数据的二维空间映射到图形的平面。默认情况下，ggplot2使用笛卡尔坐标系（coord_cartesian），但你也可以使用其他坐标系，如极坐标（coord_polar）或地图投影（coord_map）等。
  6. 分面（Facets）：分面允许你根据一个或多个变量将图形分割成多个子图形。这使得你可以在同一个图形中展示多个子集或组别的数据。分面有两种主要类型：facet_wrap（一维分面）和facet_grid（二维分面）。

attr()和attributes()函数
attr() 函数用于获取或设置单个属性。其语法为 attr(x, which)，其中 x 是对象，which 是属性的名称。如果只提供 x 和 which，该函数将返回指定的属性。如果你还提供了第三个参数，该函数将设置指定的属性并返回修改后的对象。

attributes() 函数则用于获取或设置所有属性。其语法为 attributes(obj)，其中 obj 是对象。如果只提供 obj，该函数将返回一个名为属性名称的列表，其元素为相应的属性值。如果你还提供了一个列表作为第二个参数，该函数将设置所有的属性并返回修改后的对象。
